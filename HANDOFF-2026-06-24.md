## 5. LIVING PAPERS INTEGRATION

The living-paper D1 (70a58cb3) and paper-ingestion-queue (6754f550) are the existing infrastructure for living papers. The CMS architecture extends them:

| Existing Asset | CMS Role |
|----------------|----------|
| living-paper D1 | Becomes content_items + content_versions tables |
| paper-ingestion-queue | Becomes build-pipeline-worker trigger |
| publication-publisher skill | Embeds into build-pipeline-worker |
| pdf-builder skill | Embeds into build-pipeline-worker |
| citation-manager skill | Embeds into build-pipeline-worker validation |
| graph-api Worker | Target for index-worker auto-registration |
| qwav-research Vectorize | Target for semantic search indexing |

### Content Schema for D1

```sql
CREATE TABLE content_items (
  id TEXT PRIMARY KEY,           -- e.g., "paper-trapped-ions-v1"
  type TEXT NOT NULL,            -- 'Paper', 'Project', 'Decision', 'Dataset'
  status TEXT NOT NULL DEFAULT 'draft',  -- 'draft', 'review', 'published', 'archived'
  title TEXT NOT NULL,
  author TEXT,
  markdown_source TEXT,          -- CANONICAL source of truth
  metadata JSON,                 -- Tags, domain, DOIs, citations
  r2_path TEXT,                  -- Path to rendered assets
  pages_url TEXT,                -- Deployed URL
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now')),
  published_at TEXT
);

CREATE TABLE content_versions (
  content_id TEXT REFERENCES content_items(id),
  version INTEGER,
  markdown_source TEXT,
  rendered_html TEXT,
  commit_hash TEXT,
  created_at TEXT DEFAULT (datetime('now')),
  PRIMARY KEY (content_id, version)
);
```

### Publishing Flow for Living Papers

```
1. Author submits Markdown → POST content-api/content/submit
2. Validate citations (citation-manager) + schema validation
3. Enqueue to paper-ingestion-queue
4. build-pipeline-worker dequeues:
   a. Render HTML (HTML-PUBLICATION-PAGE template + MathJax)
   b. Render PDF (pdf-builder)
   c. Upload PDF to R2: qnfo/releases/YYYY/MM/
   d. Deploy to Pages: hub.qnfo.org/papers/{slug}/
   e. Submit to Zenodo (publication-publisher)
5. index-worker auto-updates:
   a. Knowledge Graph: POST /sync with paper node + edges
   b. Discovery Index: Update qnfo/discovery/index.json
   c. Vectorize: Index paper embeddings
6. Paper visible at hub.qnfo.org/papers/{slug}/
```
