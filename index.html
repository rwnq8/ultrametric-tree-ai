📐 Mathematical Foundations

Ordinary Metric (Euclidean / Vector / Cosine):
  d(A,C) ≤ d(A,B) + d(B,C)    [Additive — standard triangle inequality]
  → Smooth, continuous space. Points can be averaged.
  → Used by all vector databases (Pinecone, Weaviate, Milvus, etc.)

Ultrametric (Non-Archimedean / Tree):
  d(A,C) ≤ max( d(A,B), d(B,C) )    [Strong triangle inequality]
  → All triangles are isosceles with two equal longest sides.
  → Every point in an open ball is its center.
  → Balls are either disjoint or nested — never partially overlap.
  → The space is totally disconnected (no continuous paths).

Tree Distance (LCA — Lowest Common Ancestor):
  Given a rooted tree T with leaves L.
  For leaves a,b ∈ L, let lca(a,b) be their lowest common ancestor.
  Then: d(a,b) = depth( lca(a,b) ) / max_depth

p-adic Numbers (The Canonical Ultrametric Field):
  For prime p, the p-adic absolute value |x|ₚ = p^{−ordₚ(x)}
  defines an ultrametric: d(x,y) = |x − y|ₚ
  This underlies v-PuNNs and p-adic neural networks.

Key Theorem — Tree Equivalence:
  Every finite ultrametric space is isometric to the leaf set
  of a rooted tree with uniform edge lengths at each level.
  Conversely, every such tree defines an ultrametric.
  
  → Decision trees, hierarchical clusters, and taxonomies
    all implicitly define ultrametrics.

🔢 p-adic (2-adic) Distance Benchmark — Direct Ultrametric Measurement

p-adic numbers are the canonical ultrametric field. For p=2, the 2-adic absolute value |x|₂ = 2−ord₂(x) measures how many factors of 2 divide x. This creates a perfect tree structure where each bit position is a hierarchy level. v-PuNNs (2025) use this exact principle — neurons are characteristic functions of p-adic balls in ℤₚ.

Prime: p=2 (2-adic / binary)

Binary Feature Encoding (2-adic)

Each numerical feature is scaled to [0, 2¹⁶) and binary-encoded. The number of matching leading bits = 2-adic valuation ord₂(|x−y|). More matching bits = shallower LCA = higher similarity.
